# Деплой askbrows как под-сайт (awwwdde-panel)

Гайд по выкладке проекта в awwwdde-panel. Проект собран в **единый Docker-образ**
по контракту гостя панели: фронт (Vite/React) собирается в статику и раздаётся тем же
FastAPI-бэкендом, что и `/api`. Один процесс, один порт.

---

## Контракт гостя (выполнен)

| Требование панели | Как выполнено |
|---|---|
| Dockerfile в корне репозитория | [`/Dockerfile`](../Dockerfile) (multi-stage) |
| Приложение слушает порт **8080** | `uvicorn … --port 8080`, `EXPOSE 8080` |
| `GET /healthz` → 200 когда готово | эндпоинт в `backend/app/main.py` |
| БД из `DATABASE_URL` | `config.py`, авто-нормализация к asyncpg |
| Миграции/сид на старте | `CMD: alembic upgrade head && python -m app.scripts.seed && uvicorn …` |

---

## Переменные окружения (задать в панели)

| Переменная | Обяз. | Назначение |
|---|---|---|
| `DATABASE_URL` | **да** | Строка подключения к **PostgreSQL**. Принимаются `postgres://`, `postgresql://`, `postgresql+psycopg2://`, `postgresql+asyncpg://` — всё приводится к asyncpg. **SQLite не поддерживается** (нужны EXCLUDE-констрейнт, `btree_gist`, `ARRAY`, native enum). |
| `JWT_SECRET` | **да** | Секрет для подписи cookie-сессий клиентов. Длинная случайная строка. |
| `BOT_TOKEN` | да (для входа) | Токен Telegram-бота — нужен для валидации `initData` (Mini App) и Login Widget. |
| `CANCELLATION_HOURS` | нет | Порог отмены/переноса, по умолчанию `6`. |
| `CORS_ORIGINS` | нет | Не требуется: фронт и API — один origin (один контейнер). |
| `TELEGRAM_WEBHOOK_SECRET` | нет | Понадобится в Фазе 3 (webhook бота). |

Уже зашиты в образ и менять не нужно: `ENV=production`, `STATIC_DIR=/app/static`,
`UPLOADS_DIR=/app/uploads`.

**Username бота** вшивается на сборке (build-arg `VITE_BOT_USERNAME`, дефолт
`askbrows_bot`). Если панель собирает другой бот — передать `--build-arg VITE_BOT_USERNAME=...`.

> ⚠️ **Postgres обязателен.** Если панель не выдаёт БД автоматически — создать
> PostgreSQL для под-сайта и прописать её `DATABASE_URL`.

## Шаги деплоя

1. **Закоммитить и запушить** ветку в `origin` (github.com/awwwdde/wife) — панель
   собирает образ из репозитория.
   ```bash
   git add -A && git commit -m "askbrows: panel-ready image (phase 1–2)"
   git push origin main
   ```
2. В **awwwdde-panel** создать под-сайт, указав этот репозиторий (Dockerfile в корне
   подхватится автоматически).
3. Прописать переменные окружения из таблицы выше (минимум `DATABASE_URL`, `JWT_SECRET`,
   `BOT_TOKEN`).
4. Запустить сборку. На старте контейнера автоматически: применятся миграции и
   идемпотентный сид (мастер «Ангелина» + демо-услуги + график Пн–Сб).
5. Панель дождётся `GET /healthz` → 200 и отдаст под-сайт на его поддомене
   (например, `askbrows.awwwdde.art`).

---

## Проверка после деплоя

- `https://<поддомен>/healthz` → `{"status":"ok"}`.
- `https://<поддомен>/` → сайт-визитка, витрина услуг подтягивается из БД.
- `https://<поддомен>/api/services` → JSON со списком услуг (значит БД и сид ок).
- Запись (`/booking`): вход через Telegram Mini App работает; браузерный Login Widget —
  **только после** привязки домена бота в BotFather: `/setdomain` → `<поддомен>`.

---

## Что НЕ входит в этот образ

- **Telegram-бот** и **воркер напоминаний** — это отдельные процессы (Фаза 3). Сейчас
  в панель выкладывается веб-часть (сайт + запись + API). Записи создаются и хранятся,
  строки напоминаний планируются в БД, но рассылка бота подключится в Фазе 3
  (webhook в тот же FastAPI + воркер как фоновая задача/отдельный сервис).
- **Redis, S3** — пока не используются рантаймом; переменные заложены на будущее.

## Старт контейнера и диагностика

Старт устойчивый (`docker-entrypoint.sh`): **ждём БД → миграции → сид → uvicorn**.
Контейнер **не падает** на сбое БД/миграции — сервер поднимается, `/healthz` отвечает
(панель видит живой контейнер), а точная причина пишется в лог. Читаем лог под-сайта:

- `[wait_for_db] … failed: …` — БД недоступна: панель не подключила Postgres или неверный `DATABASE_URL`.
- `[start] !!! CREATE EXTENSION btree_gist` — у пользователя БД нет прав на расширение (нужно у managed-Postgres — см. ниже).
- `Application startup complete` без ошибок выше — всё ок.

## Типовые проблемы

| Симптом | Причина / решение |
|---|---|
| «Контейнер не найден» / крэш-луп | Раньше — падение на `alembic` без БД. Теперь контейнер живёт; смотреть логи `[wait_for_db]`/`[start]`. Убедиться, что к под-сайту привязан **Postgres** и задан `DATABASE_URL`. |
| В логах `CREATE EXTENSION btree_gist … permission denied` | Managed-Postgres требует прав. Расширение `btree_gist` должно быть разрешено провайдером (Timeweb обычно разрешает владельцу БД). Если нет — попросить включить `btree_gist` для базы. |
| `/api/services` → 500 | Не прошли миграции — см. лог `[start] шаг 2/3`. |
| `/api/services` → пусто | Сид не отработал — см. лог `[start] шаг 3/3`. |
| Вход через Login Widget не появляется | Не привязан домен бота в BotFather (`/setdomain`). Mini App-вход при этом работает. |
| Загруженные картинки пропали после редеплоя | `/app/uploads` эфемерна без volume — ожидаемо (Фаза с медиа добавит хранилище). |
